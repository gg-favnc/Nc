Getting Started

Welcome to Nexus Command! This guide will help you get started with setting up and using the bot on your Discord server.

Inviting the Bot

To add Nexus Command to your server:

  1. Click the "Add to Discord" button on our website
  2. Select your server from the dropdown menu
  3. Grant the necessary permissions (we recommend Administrator for full functionality)
  4. Complete the captcha verification

Basic Setup

After inviting the bot, you could run the RB_setupvm command to configure voice master.

First Commands

Try these basic commands to get started:

  • [JS/PY/RS/RB]_tool - Shows all available commands
  • [JS/PY/RS/RB]_ping - Checks bot latency
  • PY_serverinfo - Shows server information

Webhook Integration

Nexus Command provides powerful webhook integration for automation, logging, and custom messaging systems.

Security Notice: Webhook URLs should be kept private as they can be used to post messages without authentication.

Webhook Basics

Webhooks are user-defined HTTP callbacks that allow real-time communication between Discord and external services.

Creating Webhooks in Discord

  1. Go to Server Settings → Integrations → Webhooks
  2. Click New Webhook
  3. Configure:
    • Name (the display name for the webhook)
    • Channel (where messages will be sent)
    • Avatar (optional image)
  4. Copy the Webhook URL

Webhook Types

Notification Webhooks

Receive real-time alerts for server events like joins, leaves, moderation actions, and more.

Custom Bots

Create pseudo-bots with custom names and avatars for specific purposes.

Data Logging

Log server activity to external databases or analytics platforms.

Advanced Webhook Features

Rate Limiting

Discord webhooks have rate limits:

  • 5 requests per second per webhook
  • Bursts up to 10 requests allowed with proper queueing

Error Handling

Common webhook errors and solutions:

401
Invalid webhook URL - check for typos
403
Webhook deleted or permissions revoked
429
Rate limited - implement exponential backoff

Security Best Practices

Rotate webhook URLs periodically

Never commit webhook URLs to public repositories

Validate incoming webhook payloads

Integration Examples

With Nexus Command

// Sample Node.js webhook sender
const sendWebhook = async (url, content) => {
    try {
        const response = await fetch(url, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
                content: content,
                username: 'Nexus Bot',
                avatar_url: 'https://cdn.discordapp.com/avatars/1353915451650080819/a_299e10df523e84f0afcc06c4351ed166.gif'
            })
        });
        console.log('Webhook delivered successfully');
    } catch (error) {
        console.error('Webhook failed:', error);
    }
};
